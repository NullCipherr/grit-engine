# Contributing Guide

Thanks for your interest in contributing to GRIT Engine.

## Before You Start

- Read the architecture docs in `docs/en`.
- Open an issue describing your proposal or bug report.
- Keep changes focused and scoped.

## Development Flow

1. Fork/branch from the default branch
2. Implement your changes
3. Run checks:

```bash
npm run typecheck
npm run build
```

4. Open a pull request with:
- Problem statement
- Technical approach
- Risks and tradeoffs
- Validation evidence

## Coding Standards

- Keep framework-agnostic boundaries intact.
- Prefer explicit naming and typed contracts.
- Avoid introducing unnecessary dependencies.
- Document non-obvious design decisions.
